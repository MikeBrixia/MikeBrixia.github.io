What is a Behavior Tree?

The Behavior Tree it's a system that allow developers to build AI logic and behaviors in a clean and visual way. In Unity it's an asset called "BehaviorTree which can be created from the Unity Editor UI.
It is mainly used to create Artificial Intelligence logic for non-player characters

The tree execute from top-left to bottom-right and has 4 types of nodes: Action, Decorator, Composite and Service.
Each node can be created, placed and inspected in the graph by the user. User can also create their own nodes by inheriting from 1 of the 4 node types, for more info about how to create your own nodes look at the nodes reference page.

The tree uses a special component called Blackboard to store and access game data at runtime, You can think of it as the brain of the Tree, which contains user-defined keys that hold properties that can be used by the tree to make decisions. For example you could have a Blackboard key called canSeePlayer of boolean type which determines if the controlled AI can see the player or not and use it as a condition to begin or stop the execution of a given branch.

User guide

Quick guide on how to get started with the Behavior Tree System inside the Unity Editor

1. To create a new Behavior Tree asset go to Assets->Create->AI->Behavior Tree

2. Edit the asset name and confirm it's creation, once you've done that double-click on the asset to open the Behavior Tree editor and inspect the selected asset. Behavior Tree assets can be inspected in the editor by double-clicking on them.

You can also open the Behavior Tree Editor window by going to Window->AI->Behavior Tree Editor and then double-click on the desired Behavior Tree asset to display it in the graph.

3. Users can inspect the asset by selecting it in the content browser. Here the developer will be able to assign a Blackboard asset for the Tree to use. For more info about the Blackboard check the Blackboard Overview page
Devs will also be able to choose if the tree asset should update each tick or at a user-defined interval.

4. Once opened you will see that in the graph there is already a node, that's the root node, a special node which will serve as the entry point of the whole Tree. Here you can place the mouse in the the graph and right-click on your mouse to make the nodes window appear. This window will display all the nodes you can place in the graph subdivided by their type(Action, Decorator ecc...), click on the node you want to create and place it in the graph. Once in the graph you can select, inspect and drag the node freely.

5. Because of their nature and design phylosophy, Decorator and Service nodes cannot exist on their own inside the graph, but must be attached to Composite and Action nodes, for this reason to create and attach a node of those two types you must first select a node in the graph and then right-click with your mouse.
